# ☁️ Introducción a Google Cloud Platform

## Contexto
Esta práctica sirvió como primera aproximación al entorno de Google Cloud Platform, utilizando credenciales temporales proporcionadas por Qwiklabs para explorar los componentes fundamentales de la consola y los servicios básicos de la plataforma en la nube.

## Objetivos
El objetivo principal fue familiarizarse con la interfaz de Google Cloud Console, comprendiendo la estructura de proyectos, la gestión de identidades y accesos mediante IAM, y el funcionamiento de las APIs de servicios cloud.

## Actividades (con tiempos estimados)
1. Acceso y exploración inicial (10 min)
2. Gestión de proyectos (5 min)
3. Navegación por servicios (5 min)
4. Configuración IAM (5 min)
5. Habilitación de APIs (5 min)

## Desarrollo
La sesión comenzó con el acceso a la consola utilizando credenciales temporales de estudiante, donde se identificó el proyecto asignado y se contrastó con el proyecto compartido "Qwiklabs Resources". A través del menú de navegación se exploraron las categorías de servicios disponibles, desde computación hasta almacenamiento y machine learning.

En la gestión de IAM, se verificaron los permisos del rol Editor asignado y se practicó la asignación de roles al otorgar permisos de Visualizador a un usuario secundario. Posteriormente, en la biblioteca de APIs, se habilitó la API de Dialogflow para comprender el proceso de activación de servicios y se revisó su documentación integrada. El flujo de trabajo demostró cómo las APIs permiten extender las capacidades de los proyectos cloud.
